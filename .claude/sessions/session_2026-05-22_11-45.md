## Problems & Solutions

- **Problem**: Forgejo API `POST /repos/.../pulls` restituisce "The target couldn't be found" per branch con `/` nel nome
  - **Solution**: merge locale fast-forward dopo rebase + push diretto su main (con nota nel commit)
  - **Lesson**: Forgejo 15.0.1 ha un bug nell'API PR creation con branch che contengono slash. Aprire PR da UI o usare branch senza slash.

- **Problem**: `feat/docker-support` divergeva da main dopo il merge di `fix/old-format`
  - **Solution**: `git rebase main` su `feat/docker-support` → merge fast-forward pulito
  - **Lesson**: quando si lavora su più branch in parallelo, fare rebase prima del merge finale

- **Problem**: `tuxbox-registry-public` push rifiutato (non-fast-forward) + conflict in `tools.toml`
  - **Solution**: `git pull --rebase` + risoluzione manuale del conflict tenendo i campi del remote (`isolation`, `system`) + i nostri (`python`, `poetry`)
  - **Lesson**: il registry può essere aggiornato da più parti; fare sempre `git pull` prima di modificarlo

- **Problem**: PyPI aveva ancora v1.1.1 (ultime 3 release non pubblicate)
  - **Solution**: release v1.3.4 completa con GitHub Release → workflow `publish-pypi.yml` scattato automaticamente
  - **Lesson**: il workflow PyPI dipende da "GitHub Release published", non dal push del tag. Serve `gh release create` oltre al tag.
